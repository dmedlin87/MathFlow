# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in MathFlow, please report it responsibly.

**DO NOT** open a public issue for security vulnerabilities.

Instead, please email us directly at [security@mathflow.example.com](mailto:security@mathflow.example.com) (replace with actual contact if available) or DM the repository maintainer.

Please include:

- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact.

We will acknowledge your report within 48 hours and work with you to resolve the issue.

## Data Privacy

MathFlow is designed to be a privacy-conscious application.

- We minimize data collection.
- We do not store personally identifiable information (PII) of students by default in this open-source version.
- Any AI integration should be configured to handle data according to relevant privacy laws (e.g., COPPA, GDPR).
